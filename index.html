SIP Trunking New
Enable SIP Trunking
Unlocks SIP Trunk Management on the admin panel and trunk configuration screens in the client portal. Works independently of Remediation.
When enabled, the following will appear
Admin: SIP Trunk Management module
Client portal: Trunk configuration wizard
Client portal: Trunk management dashboard
Remediation Process New
Enable Remediation Process
Activates the full number remediation workflow — batch management, ANI monitoring, and 90-day lifecycle — on both admin and client portals.
SIP Trunking must be enabled first. Enable SIP Trunking →
When enabled, the following will appear
Admin: Remediation Operations module (Batches, Numbers, ANI)
Client portal: Full remediation workflow
Agreements: LOA becomes active & required
Products: Remediation SKUs activated in catalog
Disable SIP Trunking?
Remediation is currently active and depends on SIP Trunking. Disabling SIP will also disable Remediation, hiding both modules from admin and client portals.

No existing data will be deleted. Both features can be re-enabled at any time.
